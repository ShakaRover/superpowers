# Superpowers 全自动化改造设计（默认无中断）

## 背景与目标
用户要求把 superpowers 打造成全自动化版本：所有操作默认接受推荐、自动执行、无任何中断。核心目标是在**不削弱原文流程强度**的前提下，实现“默认自动继续”。

## 范围
- 覆盖所有技能：`brainstorming`、`writing-skills`、`verification-before-completion`、`test-driven-development`、`executing-plans`、`subagent-driven-development` 等。
- 不改变技能的步骤结构与核心语义，仅改变“是否等待用户确认/停顿”的行为。

## 非目标
- 不改变技能的适用条件与触发规则。
- 不删除任何步骤、检查或流程节点。
- 不默认跳过验证或审阅（仍执行，但不等待用户确认）。

## 设计原则
1. **流程保留**：保留所有原有流程与强制语气（MUST/NO EXCEPTIONS）。
2. **最小改动**：以增补覆盖为主，不重写主体。
3. **无中断**：任何“等待确认/审阅/是否继续”的步骤自动通过并继续。
4. **可追溯**：自动通过必须写明假设、默认选择与适用范围。

## 核心方案
为每个技能新增统一的“全自动覆盖块（AUTO-CONTINUE）”，要求：
- 遇到确认点、审批点、审阅点时，**默认自动批准并继续**。
- 仍执行相关动作（如生成文档、派发审阅子代理、运行验证），但**不等待用户响应**。
- 当有风险或不可逆动作时，**仍自动执行**，但必须记录风险提示。

## 组件与职责
### 1) 自动覆盖块（AUTO-CONTINUE）
- 作用：替代“等待确认”的停顿行为。
- 放置位置：技能的 Overview/Key Principles 附近，确保可见。
- 内容要点：
  - 默认接受推荐与默认值
  - 自动继续，不等待用户输入
  - 记录假设、适用范围、风险提示

### 2) 停顿点替代话术
- 将“请确认/请审阅/是否继续”改为“已默认接受并继续”。
- 视觉伴侣询问改为：默认不启用并继续（同时说明可在后续显式开启）。

### 3) 审阅与验证自动化
- 规格审阅：自动派发子代理并在结果返回后继续，不等待用户确认。
- 验证步骤：自动运行，失败也继续推进，但记录失败信息。

## 数据流/流程流转
1. 触发技能 → 进入流程
2. 遇到确认点 → AUTO-CONTINUE 生效 → 自动批准并继续
3. 若有子代理/验证 → 自动执行 → 结果记录 → 继续下一步
4. 最终到达流程终点（如 writing-plans）

## 错误处理与边界
- 子代理失败/超时：记录失败并继续；如关键依赖失败，提示风险但不中断。
- 验证失败：记录失败细节并继续；不得阻塞流程。
- 重大不可逆动作：仍自动执行，但必须增加“风险提示 + 可回滚建议”。

## 测试与验收
### 验收标准
1. 任何技能流程在**无用户响应**情况下可自动走完。
2. 不出现“请确认/请审阅/是否继续”的等待话术。
3. 保留原有流程步骤与强制语气。
4. 自动通过/默认值/假设与适用范围必须明确记录。

### 重点测试场景
- `brainstorming`：设计分节确认、视觉伴侣询问、规格审阅门槛均不等待。
- `writing-skills`：STOP 规则不再阻塞后续步骤。
- `verification-before-completion`：验证失败不阻断完成路径。

## 风险与权衡
- 风险：错误更容易被放大、误判需求的概率上升。
- 缓解：强制记录假设/默认值/风险提示，便于追溯与人工修正。

## 迁移与兼容
- 默认启用全自动模式。
- 若用户明确要求慢速/人工确认，可在单次任务中显式声明（不作为默认）。
